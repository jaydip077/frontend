 ## How is it difference from other javascript framework and libraries?
 # 1.React is a library:

    React is a library not a framework. this is the most fundamental difference. A framework provides a complete opinionated structure for your application. it dictates how you handle routing data management and more.
    React is a library that provides a set of tools and components to build user interfaces.

#  2.The virtual DOM:

    React uses a virtual DOM to update and render components efficiently.
    The virtual DOM is a JavaScript object tree that represents the actual DOM.
    When you update a component, React first updates the virtual DOM. then it compares the virtual DOM with the actual DOM and only updates the parts that have changed.

 # 3.Component-based architecture:

    React is a component-based architecture. this means that your application is built from components.
    Each component is a self-contained unit of code that handles its own logic and rendering.
    Components can be reused in different parts of your application.
    This makes it easier to maintain and scale your application.

 # 4.JSX(javaScript XML):

    React uses JSX to write components.
    a syntax extenction for JavaScript that allows you to write HTML like code directly within your JavaScript files.
    While it  might look like HTML it is compiled down to regular JavaScript by a tool like Babel.this allows you to leverage the full poer of JavaScript within you UI markup making component creation more intuitive and declarative.
